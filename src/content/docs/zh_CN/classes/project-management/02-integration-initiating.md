# 📚 第二部分：项目集成管理、选择与启动 (Integration & Initiating)

### 1. 项目集成管理过程 (Project Integration Management)

集成管理是协调和整合所有其他九大知识领域、统筹项目全局的“粘合剂”：

1. **Develop Project Charter (制定项目章程)**：正式授权项目存在 [产出：**Project Charter (项目章程)**]。
2. **Develop Project Management Plan (制定项目管理计划)**：定义、编制和协调所有子计划 [产出：**Project Management Plan (项目管理计划/PMP)**]。
3. **Direct and Manage Project Work (指导与管理项目工作)**：执行项目管理计划中定义的工作 [产出：**Deliverables (可交付成果)**、**Work Performance Data (工作绩效数据)**]。
4. **Manage Project Knowledge (管理项目知识)**：利用现有知识并创造新知识，以实现项目目标。
5. **Monitor and Control Project Work (监控项目工作)**：跟踪、审查和报告整体项目进展 [产出：**Work Performance Reports (工作绩效报告)**]。
6. **Perform Integrated Change Control (实施整体变更控制)**：审查所有变更请求，批准或拒绝变更 [产出：**Approved Change Requests (批准的变更请求)**]。
7. **Close Project/Phase (结束项目或阶段)**：正式结束项目、阶段或合同 [产出：**Product Transition (产品/服务移交)**、**Lessons-learned register (经验教训登记册)**]。

---

### 2. 核心启动文件 (Initiating Documents)

* **Business Case (商业论证)**：A document commonly used to create the project charter; it justifies the investment and defines the project boundaries based on market demand, organizational need, or customer requests.（从财务和战略角度论证项目投资价值的文件，常用于作为项目章程的输入。）
* **Project Charter (项目章程)**：A document that formally recognizes the existence of a project and provides direction on the project’s objectives. It authorizes the project manager to use organizational resources.（**正式承认项目存在**，并赋予项目经理动用公司人财物权力的关键法律文件。未签字盖章前，项目在逻辑和法律上不成立。）
* **Organizational Process Assets (OPA, 组织过程资产)**：Plans, processes, policies, procedures, and knowledge bases specific to and used by the performing organization.（组织特有的计划、流程、政策、模板、历史数据和经验教训库，属于项目内部的无形资产。）

### 3. 组织结构 (Organizational Structures) 的类型与项目影响

项目经理在不同组织结构中所获得的权力和资源调配能力存在本质差异：

* **Functional (职能型组织)**：人员按专业部门划分（如销售部、技术部）。
  * *PM 特点*：项目经理几乎没有权力（Little or none），仅充当兼职的协调员（Part-time coordinator）；预算由职能部门经理（Functional Manager）控制。
* **Project-oriented (项目型组织)**：人员完全按项目组建。
  * *PM 特点*：项目经理拥有极高到完全的控制权（High to almost total），全职管理项目；项目结束后团队成员面临无处可去的风险。
* **Matrix (矩阵型组织)**：职能型和项目型的混合体，人员同时汇报给职能经理和项目经理。
  * **Weak Matrix (弱矩阵)**：偏向职能型，项目经理的权力有限，多为项目联络员。
  * **Balanced Matrix (平衡矩阵)**：项目经理与职能经理权力平分秋色，预算共同控制。
  * **Strong Matrix (强矩阵)**：偏向项目型，项目经理全职，拥有中到高度权力（Moderate to high），且有全职的项目管理行政支持人员。
