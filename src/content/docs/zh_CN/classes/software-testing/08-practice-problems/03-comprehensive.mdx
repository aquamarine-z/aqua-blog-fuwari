### 题 1：等价类划分测试（对应作业 1）

#### 题目背景
某商业银行信用卡的额度审核规则如下：
*   **额度计算公式**：$\text{综合得分} = \text{年龄得分} + \text{学历得分} + \text{婚姻得分} + \text{工龄得分}$
*   **额度评定**：若综合得分在 $12$ 点及以上，则授信额度为 $50,000$ 元；若综合得分在 $12$ 点以下，则授信额度为 $10,000$ 元。
*   各维度计分规则如下表所示：

| 年龄（岁） | 得分 | 学历 | 得分 | 婚姻状态 | 得分 | 现单位工龄 | 得分 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| $22 \sim 35$ | $5$ 点 | 硕士及以上 | $5$ 点 | 已婚 | $4$ 点 | 满 $5$ 年 | 加 $1$ 点（最多加$1$点） |
| $36 \sim 50$ | $4$ 点 | 本科 | $3$ 点 | 未婚 | $2$ 点 | 未满 $5$ 年 | $0$ 点 |
| 其他年龄 | $2$ 点 | 大专及以下 | $1$ 点 | | | | |

假设信用卡申请的输入数据包括：**年龄**（整数）、**学历**（字符串，可取：“硕士及以上”、“本科”、“大专及以下”）、**婚姻状态**（字符串，可取：“已婚”、“未婚”）、**工龄**（实数，单位：年）。请写出该系统的等价类划分，并设计测试用例。

---

#### 详细题解

##### 1. 划分输入条件的等价类

我们将各个输入参数分别划分为有效等价类与无效等价类。

| 输入条件 | 有效等价类 | 无效等价类 |
| :--- | :--- | :--- |
| **年龄** | (1) $22 \le \text{年龄} \le 35$ (5点)<br />(2) $36 \le \text{年龄} \le 50$ (4点)<br />(3) $\text{年龄} < 22$ 或 $\text{年龄} > 50$ (2点) | (4) 非整数、包含非法字符<br />(5) 空值 |
| **学历** | (6) “硕士及以上” (5点)<br />(7) “本科” (3点)<br />(8) “大专及以下” (1点) | (9) 拼写错误或不存在的选项 |
| **婚姻状态** | (10) “已婚” (4点)<br />(11) “未婚” (2点) | (12) 拼写错误或不存在的选项 |
| **工龄** | (13) $\text{工龄} \ge 5$ 门限 (加1点)<br />(14) $0 \le \text{工龄} < 5$ (0点) | (15) 负数（如 $\text{工龄} < 0$）<br />(16) 非数值类型 |

##### 2. 设计等价类测试用例

为覆盖上述等价类，我们设计如下测试用例（同时验证得分及额度的计算）：

###### (1) 有效等价类测试用例（使用弱一般等价类方法进行组合）

*   **用例 1**：覆盖 (1), (6), (10), (13)
    *   **输入**：年龄 = $28$，学历 = “硕士及以上”，婚姻 = “已婚”，工龄 = $6$
    *   **得分计算**：$5 (\text{年龄}) + 5 (\text{学历}) + 4 (\text{婚姻}) + 1 (\text{工龄}) = 15$ 点
    *   **预期输出**：综合得分 = $15$ 点，信用卡授信额度 = $50,000$ 元
*   **用例 2**：覆盖 (2), (7), (11), (14)
    *   **输入**：年龄 = $40$，学历 = “本科”，婚姻 = “未婚”，工龄 = $3$
    *   **得分计算**：$4 (\text{年龄}) + 3 (\text{学历}) + 2 (\text{婚姻}) + 0 (\text{工龄}) = 9$ 点
    *   **预期输出**：综合得分 = $9$ 点，信用卡授信额度 = $10,000$ 元
*   **用例 3**：覆盖 (3), (8), (11), (14)
    *   **输入**：年龄 = $18$，学历 = “大专及以下”，婚姻 = “未婚”，工龄 = $1$
    *   **得分计算**：$2 (\text{年龄}) + 1 (\text{学历}) + 2 (\text{婚姻}) + 0 (\text{工龄}) = 5$ 点
    *   **预期输出**：综合得分 = $5$ 点，信用卡授信额度 = $10,000$ 元

###### (2) 无效等价类测试用例（每次只测试一个无效输入）

*   **用例 4**（针对非整数年龄，覆盖等价类 4）：
    *   **输入**：年龄 = `twenty-five`，学历 = “本科”，婚姻 = “已婚”，工龄 = $6$
    *   **预期输出**：系统提示“年龄输入格式不正确，请输入正整数”。
*   **用例 5**（针对无效学历，覆盖等价类 9）：
    *   **输入**：年龄 = $30$，学历 = “高中”，婚姻 = “已婚”，工龄 = $6$
    *   **预期输出**：系统提示“学历选项不符合系统要求”。
*   **用例 6**（针对无效工龄，覆盖等价类 15）：
    *   **输入**：年龄 = $30$，学历 = “本科”，婚姻 = “已婚”，工龄 = $-1$
    *   **预期输出**：系统提示“工龄不能为负数”。

### 题 2：边界值分析法（对应作业 2）

#### 题目背景
现需要测试一个在线数值校验函数，该函数接收两个参数 $a$ 和 $b$，要求计算它们的乘积 $a \times b$。
*   $a$ 的有效取值范围为 $[10, 50]$ 之间的整数。
*   $b$ 的有效取值范围为 $[20, 80]$ 之间的整数。

请使用**健壮边界值分析法 (Robust Boundary Value Analysis)**，列出两个输入变量的边界，并设计出完整的测试用例集合。

---

#### 详细题解

##### 1. 确定边界值

根据健壮性测试方法，对每个变量我们关注 7 个测试点：
略小于最小值（$min-$）、最小值（$min$）、略大于最小值（$min+$）、正常值（$nom$）、略小于最大值（$max-$）、最大值（$max$）、略大于最大值（$max+$）。

*   **变量 $a$** 的范围为 $[10, 50]$：
    *   $min- = 9$
    *   $min = 10$
    *   $min+ = 11$
    *   $nom = 30$
    *   $max- = 49$
    *   $max = 50$
    *   $max+ = 51$
*   **变量 $b$** 的范围为 $[20, 80]$：
    *   $min- = 19$
    *   $min = 20$
    *   $min+ = 21$
    *   $nom = 50$
    *   $max- = 79$
    *   $max = 80$
    *   $max+ = 81$

##### 2. 设计健壮边界值测试用例

在单缺陷假设下，健壮性边界值分析要求设计 $6n+1$ 个用例（其中 $n$ 为变量个数，这里 $n=2$，因此共有 $13$ 个用例）。
我们将其中一个变量保持为正常值 ($nom$)，让另一个变量遍历其 7 个测试点（除去重复的两个正常值点 $(nom, nom)$）。

| 用例编号 | 变量 $a$ | 变量 $b$ | 用例设计说明 | 预期输出结果 |
| :---: | :--- | :--- | :--- | :--- |
| **1** | $30$ ($nom$) | $50$ ($nom$) | 两个变量均为正常值 | 输出 $1500$ |
| **2** | $10$ ($min$) | $50$ ($nom$) | 变量 $a$ 取最小值 | 输出 $500$ |
| **3** | $11$ ($min+$) | $50$ ($nom$) | 变量 $a$ 略大于最小值 | 输出 $550$ |
| **4** | $49$ ($max-$) | $50$ ($nom$) | 变量 $a$ 略小于最大值 | 输出 $2450$ |
| **5** | $50$ ($max$) | $50$ ($nom$) | 变量 $a$ 取最大值 | 输出 $2500$ |
| **6** | $9$ ($min-$) | $50$ ($nom$) | **【越界】** 变量 $a$ 略小于最小值 | 提示错误：“变量 $a$ 超出范围” |
| **7** | $51$ ($max+$) | $50$ ($nom$) | **【越界】** 变量 $a$ 略大于最大值 | 提示错误：“变量 $a$ 超出范围” |
| **8** | $30$ ($nom$) | $20$ ($min$) | 变量 $b$ 取最小值 | 输出 $600$ |
| **9** | $30$ ($nom$) | $21$ ($min+$) | 变量 $b$ 略大于最小值 | 输出 $630$ |
| **10** | $30$ ($nom$) | $79$ ($max-$) | 变量 $b$ 略小于最大值 | 输出 $2370$ |
| **11** | $30$ ($nom$) | $80$ ($max$) | 变量 $b$ 取最大值 | 输出 $2400$ |
| **12** | $30$ ($nom$) | $19$ ($min-$) | **【越界】** 变量 $b$ 略小于最小值 | 提示错误：“变量 $b$ 超出范围” |
| **13** | $30$ ($nom$) | $81$ ($max+$) | **【越界】** 变量 $b$ 略大于最大值 | 提示错误：“变量 $b$ 超出范围” |

### 题 3：因果图和判定表（对应作业 3）

#### 题目背景
某剧院自动售票机软件的规格说明如下：
单张电影票价格为 $10$ 元。售票机只接受 $10$ 元纸币或 $20$ 元纸币。用户投入纸币后并按下“确认购票”按钮，系统将送出电影票。
*   若售票机**有零钱找**，在送出电影票的同时退还 $10$ 元零钱。此时系统界面上“请投零钱”的黄灯是灭的。
*   若售票机**没有零钱找**，则界面上“请投零钱”的黄灯亮起。若在此状态下投入 $20$ 元纸币并按下购票按钮，系统将不送出电影票，并且退回 $20$ 元纸币（因为无法找零）；若在此状态下投入的是 $10$ 元纸币，则可以正常送出电影票（无需找零）。

请根据上述规格说明构造因果图，并用判定表设计测试用例。

---

#### 详细题解

##### 1. 建立原因和结果的映射关系

###### 原因 (Causes)
*   $c_1$：投入了 $10$ 元纸币。
*   $c_2$：投入了 $20$ 元纸币。
*   $c_3$：按下了“确认购票”按钮。
*   $c_4$：售票机**没有**零钱找（“请投零钱”黄灯亮）。

###### 结果 (Effects)
*   $e_1$：送出电影票。
*   $e_2$：退还 $10$ 元零钱。
*   $e_3$：退回输入的 $20$ 元纸币。
*   $e_4$：退回输入的 $10$ 元纸币（属于隐式异常处理）。

*注：$c_1$ 与 $c_2$ 互斥（不可同时发生）。*

##### 2. 因果关系分析及逻辑表达

*   **出票 ($e_1$)**：
    *   若投 $10$ 元且按下确认，无论是否有零钱，都可以正常出票：$(c_1 \wedge c_3)$。
    *   若投 $20$ 元、按下确认且**有零钱**（即 $\neg c_4$）时，可以出票：$(c_2 \wedge c_3 \wedge \neg c_4)$。
    *   因此，$e_1 = (c_1 \wedge c_3) \vee (c_2 \wedge c_3 \wedge \neg c_4)$。
*   **找零 ($e_2$)**：
    *   投 $20$ 元、按下确认且**有零钱**（$\neg c_4$）时，找零 $10$ 元：$e_2 = c_2 \wedge c_3 \wedge \neg c_4$。
*   **退还 $20$ 元 ($e_3$)**：
    *   投 $20$ 元、按下确认且**无零钱**（$c_4$）时，退款：$e_3 = c_2 \wedge c_3 \wedge c_4$。

##### 3. 构造判定表 (Decision Table)

由于 $c_1$ 与 $c_2$ 互斥，我们只测试有效的输入动作组合：

| 规则 | R1 | R2 | R3 | R4 | R5 | R6 |
| :--- | :---: | :---: | :---: | :---: | :---: | :---: |
| **条件（原因）** | | | | | | |
| $c_1$：投入 $10$ 元 | $1$ | $1$ | $0$ | $0$ | $0$ | $0$ |
| $c_2$：投入 $20$ 元 | $0$ | $0$ | $1$ | $1$ | $0$ | $0$ |
| $c_3$：按下购票按钮 | $1$ | $1$ | $1$ | $1$ | $0$ | $1$ |
| $c_4$：售票机**无**零钱 | $0$ | $1$ | $0$ | $1$ | $0$ | $0$ |
| **动作（结果）** | | | | | | |
| $e_1$：送出电影票 | $1$ | $1$ | $1$ | $0$ | $0$ | $0$ |
| $e_2$：退还 $10$ 元 | $0$ | $0$ | $1$ | $0$ | $0$ | $0$ |
| $e_3$：退回 $20$ 元 | $0$ | $0$ | $0$ | $1$ | $0$ | $0$ |
| $e_4$：退回 $10$ 元 | $0$ | $0$ | $0$ | $0$ | $0$ | $0$ |

##### 4. 转换测试用例

每一列规则对应一个独立的测试用例：

*   **用例 1 (对应 R1)**：投入 $10$ 元，按下确认按钮，系统**有**零钱找。
    *   *预期输出*：正常送出电影票。
*   **用例 2 (对应 R2)**：投入 $10$ 元，按下确认按钮，系统**无**零钱找。
    *   *预期输出*：正常送出电影票（不需找零）。
*   **用例 3 (对应 R3)**：投入 $20$ 元，按下确认按钮，系统**有**零钱找。
    *   *预期输出*：正常送出电影票，退还 $10$ 元零钱。
*   **用例 4 (对应 R4)**：投入 $20$ 元，按下确认按钮，系统**无**零钱找。
    *   *预期输出*：**不出票**，退回投币 $20$ 元。
*   **用例 5 (对应 R5)**：不投币，不按按钮。
    *   *预期输出*：无动作。
*   **用例 6 (对应 R6)**：不投币，直接按下确认按钮。
    *   *预期输出*：无动作，或提示“请先投入纸币”。

### 题 4：黑盒测试综合（对应作业 4）

#### 题目背景
现有一图书管理系统的“新读者注册”模块，界面包含两个输入：
1.  **读者姓名**：长度要求在 $2 \sim 10$ 个字符之间，只能包含英文字母和空格，且必须以英文字母开头。
2.  **办卡押金**：必须是区间 $[100, 300]$ 之间的整百数（即只能输入：$100$、$200$、$300$）。

请综合采用**等价类划分法**与**边界值分析法**为该新读者注册模块设计测试用例。

---

#### 详细题解

##### 1. 划分等价类与边界值分析

###### 条件 1：读者姓名
*   **有效等价类**：
    *   $E_1$：长度在 $[2, 10]$ 之间，仅含英文字母和空格，且以英文字母开头。
*   **无效等价类**：
    *   $E_2$：长度小于 $2$ 个字符（边界点：长度为 $1$）。
    *   $E_3$：长度大于 $10$ 个字符（边界点：长度为 $11$）。
    *   $E_4$：不以英文字母开头（如空格开头、数字或特殊字符开头）。
    *   $E_5$：包含非字母非空格的非法字符（如数字、中文字符、特殊符号等）。

###### 条件 2：办卡押金
*   **有效等价类**：
    *   $E_6$：输入值为 $100$、$200$ 或 $300$。
*   **无效等价类**：
    *   $E_7$：小于 $100$ 的整数（边界点：$99$）。
    *   $E_8$：大于 $300$ 的整数（边界点：$301$）。
    *   $E_9$：在 $[100, 300]$ 之间但非整百的数字（如：$150$、$250$）。
    *   $E_{10}$：非数值字符（如英文、空值）。

##### 2. 设计测试用例

我们在单缺陷和边界值结合的原则下设计覆盖全面的用例表：

| 用例编号 | 输入：读者姓名 | 输入：办卡押金 | 覆盖的等价类 / 边界值点说明 | 预期输出结果 |
| :---: | :--- | :--- | :--- | :--- |
| **1** | `Jack` (长4) | $200$ | 两个输入均合法 ($E_1, E_6$) | 注册成功 |
| **2** | `A` (长1) | $200$ | **边界值/无效姓名**：姓名长度为 1 ($E_2$) | 提示错误：“姓名长度不可少于2位” |
| **3** | `Abcdefghijk` (长11) | $200$ | **边界值/无效姓名**：姓名长度为 11 ($E_3$) | 提示错误：“姓名长度不可超过10位” |
| **4** | ` Jack` | $200$ | **无效姓名**：以空格开头 ($E_4$) | 提示错误：“姓名必须以字母开头” |
| **5** | `Jack9` | $200$ | **无效姓名**：包含非法数字 ($E_5$) | 提示错误：“姓名只能包含英文字母和空格” |
| **6** | `Jack` | $99$ | **边界值/无效押金**：押金小于 100 ($E_7$) | 提示错误：“押金需为100~300的整百数” |
| **7** | `Jack` | $301$ | **边界值/无效押金**：押金大于 300 ($E_8$) | 提示错误：“押金需为100~300的整百数” |
| **8** | `Jack` | $150$ | **无效押金**：非整百整数 ($E_9$) | 提示错误：“押金需为100~300的整百数” |
| **9** | `Jack` | `abc` | **无效押金**：非数值非法输入 ($E_{10}$) | 提示错误：“请输入合法的数字金额” |

### 题 5：逻辑覆盖测试（对应作业 5）

#### 题目背景
针对以下两段独立的代码程序，请根据题目要求分别设计白盒测试用例。

##### 题目一（考查语句覆盖与判定覆盖）
```c
void ProcessScore(int score, int is_member) {
    int discount = 0;
    if (score >= 100) {
        if (is_member == 1) {
            discount = 20;
        } else {
            discount = 10;
        }
    } else {
        discount = 0;
    }
    printf("Discount = %d\n", discount);
}
```
请分别设计：
1.  满足**语句覆盖 (Statement Coverage)** 的测试用例；
2.  满足**判定覆盖 / 分支覆盖 (Decision Coverage)** 的测试用例。

##### 题目二（考查条件覆盖、判定-条件覆盖、条件组合覆盖）
```c
void CheckStatus(int age, int has_license) {
    if (age >= 18 && has_license == 1) {
        printf("Drive Allowed\n");
    } else {
        printf("Access Denied\n");
    }
}
```
请分别设计：
1.  满足**条件覆盖 (Condition Coverage)** 的测试用例；
2.  满足**判定-条件覆盖 (C/DC)** 的测试用例；
3.  满足**条件组合覆盖** 的测试用例。

---

#### 详细题解

##### 题目一：

###### 1. 语句覆盖 (Statement Coverage)
语句覆盖要求程序中每一条可执行语句都至少执行一次。
*   我们需要执行到：`discount = 20`、`discount = 10` 和 `discount = 0`。

| 用例编号 | 输入 $score$ | 输入 $is\_member$ | 执行到的主要语句 / 分支路径 | 预期输出 |
| :---: | :---: | :---: | :--- | :--- |
| **TC1** | $120$ | $1$ | 进入第一个 `if` 及内部 `if` 真分支，执行 `discount = 20` | Discount = 20 |
| **TC2** | $120$ | $0$ | 进入第一个 `if` 内部 `else` 分支，执行 `discount = 10` | Discount = 10 |
| **TC3** | $50$ | $0$ | 进入最外层 `else` 分支，执行 `discount = 0` | Discount = 0 |

###### 2. 判定覆盖 / 分支覆盖 (Decision Coverage)
程序有两个判定点：最外层判定 $score \ge 100$，内层判定 $is\_member == 1$。每个判定的“真”、“假”分支均需执行一次。
*   最外层判定：$T_1$ (`score >= 100`)，$F_1$ (`score < 100`)。
*   内层判定：$T_2$ (`is_member == 1`)，$F_2$ (`is_member != 1`)。
上述设计在满足语句覆盖的同时，已天然满足判定覆盖，TC1，TC2，TC3 即可实现完全分支覆盖。

### 题 6：基本路径测试（对应作业 6）

#### 题目背景
针对以下程序函数进行基本路径测试：

```c
int ProcessLimit(int limit, int type) {
    int temp = 0;
    while (limit > 10) {               // 节点 4: 循环控制
        if (type == 1) {               // 节点 6: 条件分支 1
            temp = limit + 100;        // 节点 8
            break;                     // 节点 9
        } else {
            if (type == 2) {           // 节点 13: 条件分支 2
                temp = temp + 10;      // 节点 15
            } else {
                temp = temp + 20;      // 节点 18
            }
        }
        limit--;                       // 节点 22
    }
    return temp;                       // 节点 24
}
```

请根据基本路径测试方法完成以下要求：
1.  画出程序的 **DD路径图（DD-Path Graph）**；
2.  计算程序的 **圈复杂度 $V(G)$**；
3.  写出程序的一组 **基本路径集 (Basis Path Set)**；
4.  根据基本路径集设计测试用例。

---

#### 详细题解

##### 1. 画出控制流图与DD路径图

首先，分析代码的执行结构，我们将顺序执行且无分支的代码块合并。定义以下控制流节点：

*   **Node A**：函数入口及初始化（行 1~3）
*   **Node B**：`while (limit > 10)` 判定（行 4）
*   **Node C**：`if (type == 1)` 判定（行 6）
*   **Node D**：`temp = limit + 100; break;` 语句块（行 8~10），该分支执行后会跳出循环，直达 Node H
*   **Node E**：`if (type == 2)` 判定（行 13）
*   **Node F**：`temp = temp + 10;` 语句（行 15）
*   **Node G**：`temp = temp + 20;` 语句（行 18）
*   **Node I**：`limit--;` 以及回到 `while` 的控制流（行 22~23）
*   **Node H**：`return temp;`（行 24）以及函数结束

###### DD路径控制流关系：
*   $A \rightarrow B$
*   $B \rightarrow C$ （当 `limit > 10` 为真）
*   $B \rightarrow H$ （当 `limit > 10` 为假，循环结束）
*   $C \rightarrow D$ （当 `type == 1` 为真）
*   $C \rightarrow E$ （当 `type == 1` 为假）
*   $D \rightarrow H$ （由于 `break` 语句，直接跳出）
*   $E \rightarrow F$ （当 `type == 2` 为真）
*   $E \rightarrow G$ （当 `type == 2` 为假）
*   $F \rightarrow I$
*   $G \rightarrow I$
*   $I \rightarrow B$ （单轮循环结束，跳回判定）

##### 2. 计算圈复杂度 $V(G)$

我们用“强连通图”转换法（即假设引入一条从终点 $H$ 到起点 $A$ 的虚拟边，使图闭合）或标准非连通公式：
*   **边数 $e$** = $11$ （分别为：$A \rightarrow B$, $B \rightarrow H$, $B \rightarrow C$, $C \rightarrow D$, $C \rightarrow E$, $D \rightarrow H$, $E \rightarrow F$, $E \rightarrow G$, $F \rightarrow I$, $G \rightarrow I$, $I \rightarrow B$）
*   **节点数 $n$** = $9$ （分别为：$A$, $B$, $C$, $D$, $E$, $F$, $G$, $I$, $H$）

使用非强连通公式（单入口单出口）：
$$V(G) = e - n + 2 = 11 - 9 + 2 = 4$$

所以，程序的圈复杂度为 **$4$**。这意味着我们一共有 $4$ 条基本路径。

##### 3. 构建基本路径集

由于基本路径集不唯一，我们先设定一条最简路径（通常是不进入循环的路径）为基线路径，然后根据圈复杂度，每次改变一个判定的方向来寻找其他无关路径。

*   **基线路径 $P_1$**：直接不满足循环条件。
    *   路径：$A \rightarrow B \rightarrow H$
*   **路径 $P_2$**：进入循环，满足 `type == 1`，通过 `break` 跳出。
    *   路径：$A \rightarrow B \rightarrow C \rightarrow D \rightarrow H$
*   **路径 $P_3$**：进入循环，不满足 `type == 1`，满足 `type == 2`，结束本轮循环并退出。
    *   路径：$A \rightarrow B \rightarrow C \rightarrow E \rightarrow F \rightarrow I \rightarrow B \rightarrow H$
*   **路径 $P_4$**：进入循环，不满足 `type == 1`，不满足 `type == 2`，结束本轮循环并退出。
    *   路径：$A \rightarrow B \rightarrow C \rightarrow E \rightarrow G \rightarrow I \rightarrow B \rightarrow H$
